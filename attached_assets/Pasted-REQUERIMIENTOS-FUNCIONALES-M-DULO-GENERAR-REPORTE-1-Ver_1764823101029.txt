REQUERIMIENTOS FUNCIONALES – MÓDULO: GENERAR REPORTE
1. Verificar contenido y detonar la ejecución del reporte

RF1.1 El sistema deberá validar que toda la información requerida para generar un reporte esté completa.
RF1.2 El sistema deberá mostrar mensajes de advertencia si falta información obligatoria.
RF1.3 Una vez completada y validada la información, el sistema deberá permitir iniciar la generación del reporte.
RF1.4 El sistema no deberá permitir la generación del reporte si existe información faltante o inconsistente.

2. Calcular Calificación
2.1 Calificaciones por evento y generales

RF2.1 El sistema deberá calcular la calificación correspondiente a cada evento evaluado durante el entrenamiento.
RF2.2 El sistema deberá calcular la calificación general del entrenamiento con base en los criterios evaluados.
RF2.3 El sistema deberá almacenar las calificaciones individuales y generales para consulta posterior.

2.2 Lógica por reprobación y penalización del segundo intento

RF2.4 El sistema deberá identificar si un criterio de evaluación fue Reprobado en el primer intento.
RF2.5 Si el criterio fue reprobado inicialmente, el sistema deberá aplicar una penalización al segundo intento.
RF2.6 La penalización consistirá en descontar 0.30 puntos del valor nominal del criterio en el segundo intento.
RF2.7 El sistema deberá recalcular la calificación del criterio considerando el valor nominal reducido.
RF2.8 El sistema deberá recalcular la calificación general del entrenamiento si aplica penalización en uno o más criterios.
RF2.9 El sistema deberá registrar en el historial del entrenamiento que se aplicó una penalización por reprobación.
RF2.10 El sistema deberá mostrar en el reporte una marca o leyenda indicando que ese criterio tuvo penalización.
RF2.11 La penalización solo será aplicable al segundo intento. Intentos adicionales no deberán ser penalizados, salvo nueva regla futura.

3. Generar Gráfica de Radar

RF3.1 El sistema deberá generar una gráfica de radar individual al finalizar cada entrenamiento.
RF3.2 La gráfica deberá representar los criterios evaluados y su desempeño.
RF3.3 El sistema deberá permitir incluir la gráfica individual dentro del reporte final.
RF3.4 Al finalizar el ciclo de entrenamiento, el sistema deberá generar una gráfica de radar grupal, comparando promedios o resultados de varios operadores.
RF3.5 El sistema deberá almacenar las gráficas en el historial del trabajador y en los registros estadísticos.

4. Generar registros para estadística de entrenamientos

RF4.1 El sistema deberá generar automáticamente un registro estadístico por cada entrenamiento realizado.
RF4.2 El sistema deberá almacenar información histórica por trabajador (historial individual).
RF4.3 Los registros deberán ser accesibles para análisis por supervisores, jefes inmediatos o administradores.
RF4.4 Los registros estadísticos deberán alimentar dashboards o reportes globales del sistema.

5. Botón para Enviar el Reporte

RF5.1 El sistema deberá incluir un botón que permita enviar el reporte final generado.
RF5.2 El sistema deberá permitir enviar el reporte al jefe inmediato del trabajador.
RF5.3 El sistema deberá permitir enviar el reporte al operador (estudiante/entrenando).
RF5.4 El sistema deberá registrar fecha, hora y destinatarios del envío.
RF5.5 El sistema deberá permitir reenviar reportes generados previamente.

6. Mostrar avance de programa en pantalla inicial

RF6.1 El sistema deberá mostrar el avance del programa de entrenamiento en la pantalla inicial del usuario.
RF6.2 El avance deberá actualizarse automáticamente al completarse un entrenamiento.
RF6.3 El avance deberá representarse mediante porcentajes, barras de progreso o indicadores visuales.
RF6.4 El sistema deberá permitir consultar detalles del avance (criterios evaluados, sesiones, horas acumuladas).

7. Actualizar registros de calidad

RF7.1 El sistema deberá generar un registro de calidad por cada entrenamiento realizado.
RF7.2 El sistema deberá almacenar las horas de entrenamiento realizadas por el trabajador.
RF7.3 El sistema deberá actualizar el avance del programa anual con base en un plan preconfigurado.
RF7.4 El sistema deberá generar un reporte individual de cada entrenamiento, enlazado al historial del trabajador.
RF7.5 El sistema deberá permitir consultar o exportar los registros de calidad para auditorías o revisiones internas.
